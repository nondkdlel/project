## JS & ECMAScript
```
  현재 사용하는 Javascript의 ES버전은 ES6 뿐 아니라 상위 버전들에 많은 기능들이 추가됐으니까
  알아두면 작업 하실 때 좀 더 생산성이 올라갈거에용.
  Javascript도 변수 키워드, 함수 표현방식, 호이스팅 Call by reference 등 좀 더 깊게 배우면,
  개발을 호율적으로 할 수 있을 거에영
  
  저도 끊임없이 배우려고 하고 부족한 부분은 알고리즘이나 기능 자체를 직접 만들기도 하니까
  직접 다 만들어보는 것도 좋아요
  처음부터 만들어진 기능을 사용하는 것과 스스로 만들어 본 다음 써보는 건 이해하는 것이 다르니까요.
```

## React
```
  리액트는 어느정도 잘 사용하고 있는 것 같아 보이네용
  다만 리액트도 좀더 고급기능을 쓰는 편이 웹 퍼포먼스 향상에도 도움이 되고 리액트 이해도도 많이 올라갑니다.
  특히 hook은 useCallback, useMemo, useReducer, memo 등은 알아두면 좋습니당

  그리고, 리액트는 cli로도 개발환경 구성을 편하게 할 수 있지만 필요한 기능들은 커스텀해야 하기 때문에
  typescript, webpack5, babel7, eslint, prettier는 공부하셔야 합니다.
  이 것들은 전부 개발의 생산성과 효율성 나아가서 프로덕션의 퍼포먼스까지 영향을 미칩니당

  이 외에도 여러 번들링 모듈이 많이 있으니까 취향에 맞게 도전하시면 되용
```

## Store
```
  보니까 redux를 사용했더라구요.
  마침 저희도 redux를 사용합니다. 그 전까진 redux, recoil, mobx 등 여러 상태관리 라이브러리를 도입하려 했는데,
  결국은 redux로 굳혀졌어요. 그래도 redux뿐 아니라 리액트의 상태관리 라이브러리가 어떤 프로세스를 가지고 있는지
  알아보면 redux를 이해하는데에도 도움이 될 겁니다.

  redux는 잘 사용하고 계시네용
```

## CSS
```
  CSS파일은 되도록 지양합니다.
  글로벌한 스타일이 아닌 경우엔 Css-in-Js를 사용하는 것이 좋습니다.
  CSS 파일은 페이지 전체에 영향을 미치기 때문에 초기 렌딩과 퍼포먼스 향상에 도움이 되지 않습니다.
  Css-in-Js 중 추천하는 것은 styled-components 적용을 추천해드려용
  css는 물론 scss도 지원해주니 좋아용
```
